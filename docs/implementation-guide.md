# scripta 実装手順書

各ステップは前のステップの成果物の上に積み上げる形で進める。
ステップごとに動作確認ポイントを設け、確認が通ってから次に進む。

---

## Step 1: プロジェクトの雛形作成

Tauri v2 + React + TypeScript のプロジェクトをスキャフォールドし、最低限の画面が表示されるところまで持っていく。

### やること

- `pnpm create tauri-app@latest` で Tauri v2 + React + TypeScript テンプレートを生成（パッケージマネージャは pnpm を選択）
- `pnpm install` で依存をインストール
- Biome を導入し `biome.json` を設定（`noExplicitAny` ルール有効化）
- テンプレートのデフォルト画面がそのまま動くことを確認

### 動作確認

- `pnpm tauri dev` でアプリウィンドウが起動する
- テンプレートのデフォルト画面（Vite + Tauri のロゴ等）が表示される
- `pnpm lint` が正常に実行される

---

## Step 2: UI ライブラリの導入とレイアウト骨格

Tailwind CSS、zustand 等を導入し、アプリの基本レイアウトを組む。

### やること

- Tailwind CSS v4（`tailwindcss` + `@tailwindcss/vite`）、zustand v5、lucide-react をインストール
- CSS 変数によるダーク/ライトテーマの定義
- 2 ペインレイアウトのコンポーネントを作成:
  - `TabBar` — タブバー兼タイトルバー（ドラッグ領域、macOS ネイティブトラフィックライト使用）
  - `Sidebar` — 左サイドバー（この時点では空でOK）
  - エディタ領域のプレースホルダー
  - `StatusBar` — 下部ステータスバー
- テーマ切替ボタンの設置（ステータスバー等に）
- Tauri のウィンドウ設定で `titleBarStyle: "Overlay"` + `hiddenTitle: true` を設定

### 動作確認

- アプリにタブバー（ドラッグ領域兼用）、サイドバー、メイン領域、ステータスバーが表示される
- タブバーをドラッグしてウィンドウを移動できる
- テーマ切替ボタンでダーク/ライトが切り替わり、全体に反映される

---

## Step 3: Rust 側のファイル読み書きコマンド

最小限のファイル操作コマンドを Rust 側に実装し、フロントエンドから呼び出せるようにする。

### やること

- Rust 側に `commands/file.rs` を作成
  - `read_file(path)` — ファイル内容を文字列で返す
  - `write_file(path, content)` — ファイルに書き込む
- `lib.rs` にコマンドを登録
- フロントエンド側に Tauri コマンドラッパー (`lib/commands.ts`) を作成
- エディタ領域に仮のテキストエリアを配置し、ハードコードしたパスのファイルを読み込み・保存する動作を確認

### 動作確認

- アプリ起動時にテスト用 `.md` ファイルの内容がテキストエリアに表示される
- テキストを編集して保存ボタンを押すとファイルに書き戻される
- ファイルシステム上で内容が更新されていることを確認

---

## Step 4: CodeMirror 6 の導入

テキストエリアを CodeMirror 6 に置き換え、Markdown のシンタックスハイライトが効くようにする。

### やること

- `@uiw/react-codemirror`、`@codemirror/lang-markdown`、`@codemirror/language-data` をインストール
- `MarkdownEditor` コンポーネントを作成（CM6 ラッパー）
- Markdown 言語拡張を設定
- Step 3 で作ったファイル読み書きを CM6 と接続

### 動作確認

- `.md` ファイルを開くと CodeMirror エディタで表示される
- Markdown 構文（`# 見出し`、`**太字**`、`` `コード` `` 等）がハイライト表示される
- 編集して保存するとファイルに反映される

---

## Step 5: オートセーブとキーボードショートカット

編集の利便性を上げるため、自動保存と `Cmd+S` を実装する。

### やること

- `useAutoSave` フック: 編集後 2 秒のデバウンスで自動保存
- `Cmd+S` キーバインドで即座に保存
- ステータスバーに保存状態（保存済み / 未保存）を表示

### 動作確認

- ファイルを編集して 2 秒待つと自動保存される
- `Cmd+S` で即座に保存される
- ステータスバーの表示が編集中は「未保存」、保存後は「保存済み」に変わる

---

## Step 6: ワークスペースとファイルツリー

フォルダを開いてファイルツリーを表示し、ファイルをクリックして開けるようにする。

### やること

- `@tauri-apps/plugin-dialog` を導入（`pnpm tauri add dialog`）
- Rust 側に `commands/workspace.rs` を作成
  - `list_directory(path)` — 指定フォルダ直下のエントリ一覧を返す（遅延読み込み用）
- フロントエンド:
  - 「フォルダを開く」ボタン / メニュー
  - `FileTree` コンポーネント（展開/折りたたみ対応）
  - ファイルクリックでエディタに内容を読み込み
- zustand ストアでワークスペースパスと開いているファイルを管理

### 動作確認

- 「フォルダを開く」でフォルダ選択ダイアログが開く
- 選択したフォルダの内容がサイドバーにツリー表示される
- フォルダをクリックすると展開され、子要素が表示される（初回クリック時に取得）
- `.md` ファイルをクリックするとエディタに内容が表示される

---

## Step 7: タブバー（複数ファイル）

複数のファイルを同時に開けるようにする。

### やること

- `TabBar` コンポーネント: 開いているファイルをタブ表示
- タブのクリックでアクティブファイルを切り替え
- タブの × ボタンで閉じる（`Cmd+W`）
- 未保存タブにインジケーター（ドット等）表示
- zustand ストアで開いているタブの一覧を管理

### 動作確認

- ファイルツリーから複数ファイルを開くとタブが増える
- タブクリックでエディタの内容が切り替わる
- `Cmd+W` でアクティブなタブが閉じる
- 未保存のタブにはインジケーターが付く

---

## Step 8: Live Preview — 見出し

Live Preview の第一歩。見出しのインライン描画を実装する。

### やること

- `live-preview/` ディレクトリを作成
- `ViewPlugin.fromClass()` でカーソル位置を監視する基盤を構築
- `headings.ts`: カーソル外の `# ` ~ `###### ` マーカーを非表示にし、フォントサイズを変更
- カーソルがある行は生 Markdown を表示する仕組みを実装

### 動作確認

- `# 見出し` と入力し、カーソルを別の行に移すと `#` が消えてフォントが大きくなる
- カーソルを見出し行に戻すと `# ` が再表示され、通常のテキストとして編集できる
- `##` ~ `######` でフォントサイズが段階的に変わる

---

## Step 9: Live Preview — 太字・斜体

### やること

- `emphasis.ts`: カーソル外の `**太字**` → `**` を非表示にし太字スタイル適用
- `*斜体*` → `*` を非表示にし斜体スタイル適用
- `***太字斜体***` への対応

### 動作確認

- `**太字**` と入力しカーソルを離すと、`**` が消えて太字で表示される
- `*斜体*` で斜体が表示される
- カーソルを戻すとマーカーが再表示される

---

## Step 10: Live Preview — リンク・画像

### やること

- `@tauri-apps/plugin-shell` を導入（`pnpm tauri add shell`）— リンクを外部ブラウザで開くために使用
- `links.ts`: `[text](url)` をクリッカブルなリンクとして描画（`Decoration.widget`）。クリック時は `plugin-shell` の `open` で外部ブラウザに開く
- `images.ts`: `![alt](path)` の下に画像をインライン表示（`Decoration.widget`）

### 動作確認

- `[Google](https://google.com)` と書いてカーソルを離すと、リンクテキストとして描画される
- リンクをクリックすると外部ブラウザで URL が開く
- `![alt](./image.png)` と書くと画像がインライン表示される
- カーソルを当てると生 Markdown に戻る

---

## Step 11: Live Preview — コードブロック・リスト・引用・水平線

残りの Live Preview デコレーションをまとめて実装する。

### やること

- `code-blocks.ts`: フェンスドコードのシンタックスハイライト
- `lists.ts`: `- [ ]` / `- [x]` をクリッカブルなチェックボックスに
- `blockquotes.ts`: `>` 引用に左ボーダー付きスタイル
- `horizontal-rules.ts`: `---` を視覚的な水平線として描画

### 動作確認

- ` ```js ... ``` ` がシンタックスハイライト付きで表示される
- `- [ ] タスク` がチェックボックスになり、クリックで `[x]` に切り替わる
- `> 引用` に左ボーダーが付く
- `---` が水平線として描画される

---

## Step 12: ファイル操作（作成・リネーム・削除）

ファイルツリー上での操作を充実させる。

### やること

- Rust 側:
  - `create_file(path)` / `create_directory(path)`
  - `rename_entry(old_path, new_path)`
  - `delete_entry(path)` — `trash` クレートでゴミ箱移動
- フロントエンド:
  - 右クリックコンテキストメニュー（新規作成、リネーム、削除）
  - インラインリネーム入力
  - 削除確認ダイアログ

### 動作確認

- ファイルツリーで右クリック → 「新規ファイル」でファイルが作成される
- 「リネーム」でファイル名がインライン編集可能になり、Enter で確定
- 「削除」でゴミ箱に移動される（ファイルシステム上で消えていること確認）
- 開いていたファイルを削除した場合、タブが閉じる

---

## Step 13: ファイル変更監視

外部からのファイル変更をアプリに反映する。

### やること

- Rust 側: `notify` クレートでワークスペースのファイル変更を監視
- Tauri イベントでフロントエンドに通知
- フロントエンドで変更を受け取り:
  - ファイルツリーの更新
  - 開いているファイルの内容を再読み込み（編集中でなければ）

### 動作確認

- アプリ外で `.md` ファイルを編集し保存すると、エディタの内容が更新される
- アプリ外でファイルを追加すると、ファイルツリーに反映される
- アプリ外でファイルを削除すると、ファイルツリーから消える

---

## Step 14: 検索機能

全文検索とクイックオープンを実装する。

### やること

- Rust 側: `commands/search.rs`
  - `search_files(workspace, query)` — 全文検索、マッチ行を返す
  - `search_filenames(workspace, query)` — ファイル名検索
- フロントエンド:
  - `SearchPanel`: `Cmd+Shift+F` で表示、全文検索結果を一覧表示
  - `CommandPalette`: `Cmd+P` で表示、ファイル名のファジー検索
  - 検索結果クリックで該当ファイル・行にジャンプ

### 動作確認

- `Cmd+Shift+F` で検索パネルが開く
- キーワードを入力するとワークスペース内の全 `.md` ファイルからマッチが表示される
- 結果をクリックすると該当ファイルの該当行にカーソルが移動する
- `Cmd+P` でコマンドパレットが開き、ファイル名で絞り込みできる

---

## Step 15: 仕上げ

使い心地を整え、状態の永続化を行う。

### やること

- `@tauri-apps/plugin-store` を導入（`pnpm tauri add store`）— アプリ状態の永続化に使用
- ワークスペース記憶: `plugin-store` で前回開いていたフォルダパスを保存し、起動時に自動で開く
- ウィンドウ状態永続化: 位置・サイズ・最大化状態を `plugin-store` で保存・復元
- サイドバーの表示/非表示トグル（`Cmd+B`）
- エラーハンドリング:
  - ファイル読み込み失敗時のフォールバック表示
  - ファイル I/O コマンドの自動リトライ（`withRetry()` — 指数バックオフ、最大3回）
  - エラーメッセージの日本語翻訳（`translateError()` — OS エラー、ネットワークエラーのパターンマッチ）
  - オートセーブのリトライ（一時エラー時は最大3回、ステータスバーに「リトライ中...」表示）
  - リトライ最終失敗時のトースト通知
- ステータスバー情報: カーソル位置（行:列）、文字数の表示

### 動作確認

- アプリを閉じて再起動すると、前回のワークスペースが開かれる
- ウィンドウの位置・サイズが前回と同じ状態で復元される
- `Cmd+B` でサイドバーがトグルされる
- 存在しないファイルを開こうとするとエラーメッセージが表示される
- ステータスバーにカーソル位置と文字数が表示される
- 一時的なファイルエラー（ロック等）がリトライで自動回復する
- エラーメッセージが日本語で表示される
